Interview Checklist

  • 1. Explain the use of "this" in JavaScript
    Answer: The "this" keyword refers to the current context, which can vary based on function invocation.
  • 2. Explain the Prototype Chain
    Answer: JavaScript uses prototype-based inheritance, allowing objects to inherit properties from other objects.
  • 3. What are Callbacks, Promises, and Async/Await?
    Answer: Callbacks allow functions to be passed as arguments; promises handle asynchronous events, and async/await makes async code more readable.
  • 4. Explain Event Bubbling and Capturing
    Answer: Event bubbling and capturing describe the event propagation phases, controlling how events are handled in the DOM.
  • 5. What are arrow functions?
    Answer: Shorter syntax for function expressions, with no binding to 'this'.
  • 6. Explain async/await.
    Answer: Syntax for handling asynchronous operations, making code easier to read.
  • 7. What is destructuring?
    Answer: Allows extraction of properties from objects and arrays.
  • 8. How does 'this' keyword work in JavaScript?
    Answer: Refers to the object it belongs to, behavior depends on function call context.
  • 9. What are JavaScript promises?
    Answer: Handle async operations, with states: pending, fulfilled, rejected.
  • 10. Explain the concept of scope in JavaScript.
    Answer: Refers to accessibility of variables, objects, functions in code.
  • 11. Explain JavaScript callbacks.
    Answer: Function passed into another function to be executed later.
  • 12. Explain the difference between map and forEach.
    Answer: map returns a new array, forEach does not return anything.
  • 13. What is JSON and how is it used in JavaScript?
    Answer: Format for structuring data, used for data exchange between server and web app.
  • 14. What is event delegation?
    Answer: Technique to handle events at a higher level in the DOM.
  • 15. Explain JavaScript currying.
    Answer: Technique to transform a function with multiple args into a series of functions with a single argument.
  • 16. What are Higher Order Functions?
    Answer: Functions that take other functions as arguments or return them.
  • 17. How does error handling work in JavaScript?
    Answer: try...catch statement is used to handle exceptions.
  • 18. What is a generator function?
    Answer: Function that can pause execution and yield control back.
  • 19. Explain prototype chain.
    Answer: Mechanism by which JavaScript objects inherit properties from other objects.
  • 20. What is debouncing?
    Answer: Technique to limit the rate at which a function is executed.
  • 21. What is throttling?
    Answer: Limits the number of times a function can execute over time.
  • 22. What is DOM manipulation in JavaScript?
    Answer: Techniques to create, delete, and modify elements in the DOM.
  • 23. What is 'strict mode' in JavaScript?
    Answer: Enables strict syntax to catch errors and ensure cleaner code.
  • 24. What are IIFEs in JavaScript?
    Answer: Immediately Invoked Function Expressions to execute code right after defining.
